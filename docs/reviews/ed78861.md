## Comments

### P2 — Restore Auto and Local Tony to picker
**File:** `app/src/main/kotlin/com/mlainton/nova/MainActivity.kt:537`

> Filtering out `AUTO` and `LOCAL_TONY` removes the only UI path that sets these modes in `BrokerPrefs`; both modes are still supported by `backendProviderForCurrentBrain()`/`processThroughBroker()`, and the on-device status dialog still tells users to switch to Local Tony. For new users or anyone not already persisted in those modes, smart routing and offline Tony become unreachable.

**Assessment:** valid. The regression is concrete:

1. `MainActivity.kt:1556` — `showOnDeviceModelStatus()` literally tells the user *"Switch to 'Local Tony' brain mode to use it offline"* — an instruction that now points at a nonexistent picker entry.
2. `backendProviderForCurrentBrain()` still handles `AUTO -> "auto"` (backend `chat.py:254` routes `auto`/`smart`/`""` through `model_router_smart.choose_provider`) and `LOCAL_TONY -> null` (on-device fallback). Both modes work; they're just unreachable.
3. Existing users whose stored pref is `AUTO` or `LOCAL_TONY` still get those modes (BrokerPrefs deserialises the enum name). Fresh installs and anyone who switched away can never get back.

**My take (open for your call):**

Codex is right by the letter — the trim is a real regression for new users of two working features. But the commit message "trim picker to live brains" reads AUTO/LOCAL_TONY as explicitly out-of-scope: AUTO is a *router*, LOCAL_TONY is *on-device*, neither is a *brain* in the strict "provider seat" sense. If that intent is real, the on-device help dialog needs its language updated too (drop "Switch to Local Tony" → point at a dedicated on-device toggle or the settings drawer).

Two forks:
- **A. Revert just the picker trim.** Small follow-up commit; keeps the other three TARGETs. Zero-risk path.
- **B. Accept the regression as intended** and fix the stale instruction at `showOnDeviceModelStatus()` in the same follow-up, plus add a dedicated on-device toggle somewhere the user can still find.

Stopping here for direction — not merging a fix without your call.
